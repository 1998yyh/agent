🤖 DevAgent 设计文档

基于 claude-quickstarts 的多 Agent 软件开发系统

项目概述

DevAgent 是一个用于学习的、功能完备的通用软件开发多 Agent 系统。 它实现了从 需求分析 → 技术方案 → AI 编码 → 测试生成 → 交付审查 的完整开发流程。

🎯 项目目标

学习目标

  • 深入理解 Agent 架构原理
  • 掌握 Claude Tool Use 机制
  • 学习多 Agent 协作模式
  • 理解工作流编排设计

功能目标

  • 需求自动分析与文档化
  • 技术方案自动生成
  • 代码自动编写与修改
  • 测试用例自动生成与执行
  • 交付物自动审查与打包

技术目标

  • 基于 Python + Anthropic API
  • 复用 claude-quickstarts 架构
  • 支持 CLI 和 Web 两种界面
  • 模块化设计，易于扩展

核心设计理念

渐进式演进 三阶段逐步完善
→
职责分离 每个 Agent 单一职责
→
上下文传递 共享状态管理
→
可观察性 完整执行追踪

Agent 详细设计

Agent 基类设计

class BaseAgent(ABC): """所有 Agent 的抽象基类""" name: str # Agent 标识符 system_prompt: str # 系统提示词 tools: list[Tool] # 可用工具列表 async def execute(self, context: WorkflowContext) -> AgentResult: """执行 Agent 任务（抽象方法）""" pass def get_llm_client(self) -> Anthropic: """获取 Claude API 客户端""" return Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

5 个专用 Agent 职责

Agent 输入 输出 核心工具 系统提示词要点
需求分析 Agent 用户原始需求 结构化需求文档 FileWrite, Think "你是一位产品专家，擅长从模糊需求中提取明确的功能点"
技术方案 Agent 需求文档 技术设计文档 + 文件结构 FileWrite, Search, Think "你是一位架构师，负责设计清晰的技术方案"
AI 编码 Agent 设计文档 + 文件结构 源代码文件 FileWrite, Git, Shell "你是一位高级开发者，编写高质量、可维护的代码"
测试生成 Agent 源代码 测试用例 + 测试报告 FileWrite, CodeRunner "你是一位 QA 工程师，确保代码质量"
交付审查 Agent 全部产物 审查报告 + 打包 FileRead, Git, Shell "你是一位技术负责人，负责最终交付审查"

Agent 交互示例

用户：创建一个待办事项管理应用
[WorkflowEngine] 创建新工作流 workflow_001
[RequirementsAgent] 开始分析需求... → 调用 think 工具进行推理 → 调用 file_write 创建 requirements.md
工具输出：成功写入 requirements.md (520 字符)
[DesignAgent] 开始设计技术方案... → 读取 requirements.md → 调用 search 搜索最佳实践 → 调用 file_write 创建 design.md 和项目结构
[CodingAgent] 开始编码实现... → 读取 design.md → 调用 file_write 创建各个源文件 → 调用 git 初始化仓库并提交
[TestingAgent] 开始生成测试... → 分析源代码 → 创建测试文件 → 调用 code_runner 执行测试
[DeliveryAgent] 开始交付审查... → 审查代码质量 → 生成审查报告 → 创建发布包
[WorkflowEngine] 工作流完成！输出交付物。

工具系统详细设计

工具基类

class Tool(ABC): """所有工具的抽象基类""" name: str # 工具名称 description: str # 工具描述 input_schema: dict # 输入模式 (JSON Schema) def to_dict(self) -> dict: """转换为 Claude API 格式""" return { "name": self.name, "description": self.description, "input_schema": self.input_schema, } async def execute(self, **kwargs) -> str: """执行工具（抽象方法）""" raise NotImplementedError

Git 工具实现示例

class GitTool(Tool): """Git 版本控制工具""" def __init__(self): super().__init__( name="git", description="""执行 Git 操作""", input_schema={ "type": "object", "properties": { "command": { "type": "string", "description": "Git 命令 (init/add/commit/status)", "enum": ["init", "add", "commit", "status"], }, "message": { "type": "string", "description": "提交消息", }, }, "required": ["command"], }, ) async def execute(self, command: str, message: str = "") -> str: """执行 Git 命令""" cmd = ["git", command] if command == "commit" and message: cmd.extend(["-m", message]) result = subprocess.run(cmd, capture_output=True, text=True) return result.stdout or result.stderr

工作流编排设计

WorkflowEngine 核心类

class WorkflowEngine: """工作流引擎 - 核心编排逻辑""" STAGES = [ "requirements", # 需求分析 "design", # 技术方案 "coding", # AI 编码 "testing", # 测试生成 "delivery", # 交付审查 ] async def execute(self, user_request: str) -> DeliveryResult: """执行完整工作流""" context = WorkflowContext() context.user_request = user_request context.workflow_id = generate_uuid() for stage in self.STAGES: agent = self.get_agent(stage) result = await agent.execute(context) context = result.update_context(context) # 状态持久化 await self.save_checkpoint(context) # 错误处理 if result.has_error: await self.handle_error(stage, result) break return context.delivery_result

上下文存储设计

class WorkflowContext: """工作流上下文 - 在各 Agent 之间传递状态""" workflow_id: str user_request: str current_stage: str stages_completed: list[str] # 各阶段产物 requirements_doc: str = None design_doc: str = None source_files: list[str] = [] test_results: dict = None delivery_report: str = None # 元数据 created_at: str updated_at: str errors: list[str] = []

检查点持久化

每个阶段完成后，将上下文状态保存到磁盘，支持：

断点续跑（中断后恢复）
执行历史追溯
多工作流并行
审计日志
